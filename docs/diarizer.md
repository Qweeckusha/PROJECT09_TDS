# Diarizer Module
> `docs-ver: 1.0` | `Library: pyannote.audio`

## Введение
Модуль отвечает за **диаризацию аудио** - процесс сегментации записи по спикерам (Speaker Diarization). На выходе мы получаем временные интервалы с привязкой к конкретному говорящему.

### Ключевые особенности:
- **Предобработка аудио** - автоматическая конвертация в формат, совместимый с pyannote (16 кГц, моно, WAV)
- **Асинхронная загрузка модели** - пайплайн загружается один раз при создании экземпляра через `__init__` класса
- **Поддержка GPU/CPU** - автоматический выбор устройства через PyTorch, рекомендуется GPU

## Архитектура класса

### Инициализация пайплайна
```python
class CustomDiarizer:
    def __init__(self,
                 token: str = config.TOKEN,
                 model: str = config.DIARIZATION_MODEL):
        self.pipeline = Pipeline.from_pretrained(model, token=token)
        device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
        self.pipeline.to(device)
```

**Параметры инициализации:**

| Параметр | Тип | Описание                                                                   |
|----------|-----|----------------------------------------------------------------------------|
| `model` | `str` | Идентификатор модели на HuggingFace Hub                                    |
| `token` | `str` | Токен доступа HuggingFace (нужен только для обновлений и загрузки моделей) |

> [!IMPORTANT]
> Некоторые gated-модели pyannote требуют **принятия пользовательского соглашения** на HuggingFace и валидный токен. Без этого `Pipeline.from_pretrained()` вернет ошибку.


## Вспомогательный метод

### `_preprocess_audio()`
Конвертирует аудио в формат, совместимый с pyannote: **16 кГц, моно, WAV**.

```python
def _preprocess_audio(self, path: str) -> str:
```

**Причины**
- pyannote.audio обучена на данных в формате 16 кГц моно, поэтому важно синхронизировать форматы данных
- Входные файлы могут быть в любом формате (`.mp3`, `.ogg`, `.m4a`) с любой частотой дискретизации
- ffmpeg выполняет конвертацию «на лету»

**Принцип работы:**
1. Если файл уже `.wav` - возвращается без изменений
2. Иначе вызывается ffmpeg:
   ```python
   subprocess.run([
       "ffmpeg", "-i", str(path_obj),
       "-ar", "16000",    # Частота дискретизации: 16 кГц
       "-ac", "1",        # Количество каналов: 1 (моно)
       "-y",              # Перезаписать существующий файл
       str(wav_path)
   ], check=True, capture_output=True)
   ```
3. После обработки временный `.wav` файл **удаляется** в блоке `finally` метода `Diarization()`

## Основной метод

### `Diarization()`
Выполняет диаризацию аудиофайла и возвращает список сегментов со спикерами.

```python
def Diarization(self, path: str) -> list[dict]:
```

**Параметры:**
- `path` - путь к аудиофайлу (любой формат)

**Возвращает:**
```python
[
    {"start": 0.0, "end": 5.5, "speaker": "SPEAKER_00"},
    {"start": 5.5, "end": 12.3, "speaker": "SPEAKER_01"},
    {"start": 12.3, "end": 18.7, "speaker": "SPEAKER_00"},
    ...
]
```


## Принцип работы

### 1. Предобработка
```python
preprocessed_path = self._preprocess_audio(path)
```
Аудио конвертируется в 16 кГц моно WAV (если нужно).

### 2. Запуск пайплайна
```python
result = self.pipeline(preprocessed_path).speaker_diarization.itertracks(yield_label=True)
```
Пайплайн pyannote выполняет:
- **Сегментацию** - разбивает аудио на участки речи
- **Эмбеддинг спикеров** - извлекает векторные представления голоса (speaker embeddings)
- **Кластеризацию** - группирует эмбеддинги по спикерам (каждый кластер = один спикер)
- **Присвоение меток** - `SPEAKER_00`, `SPEAKER_01` и т.д.

### 3. Форматирование результата
```python
return [
    {
        "start": turn.start,
        "end": turn.end,
        "speaker": speaker,
    } for turn, _, speaker in result
]
```

### 4. Очистка
```python
finally:
    if preprocessed_path != path and Path(preprocessed_path).exists():
        Path(preprocessed_path).unlink()
```
Временный WAV-файл удаляется, если он был создан.

## Использование в системе

### Пример вызова:
```python
from src.diarizer import CustomDiarizer

diarizer = CustomDiarizer()
segments = diarizer.Diarization("audio.mp3")

for seg in segments:
    print(f"[{seg['start']:.2f} - {seg['end']:.2f}] {seg['speaker']}")
```

### Интеграция с другими модулями:
- **`Summarizer`** вызывает `diarizer.Diarization()` параллельно с транскрибацией
- Результаты объединяются в `Summarizer._align_segments()` по временным меткам
- Финальный текст содержит и слова, и спикеров

## Ограничения и известные проблемы

1. **Токен HuggingFace** - модели pyannote приватные, требуется токен и принятие соглашения
2. **Фоновый шум** - качество диаризации падает при сильном шуме или наложении голосов
3. **Один спикер** - если в аудио только один говорящий, модель всё равно может разбить его на несколько «псевдо-спикеров», но на тестах такого не случалось.


## Зависимости

- `config` - конфигурация модели и токена


## Ключевые моменты для понимания:

1. **Что такое диаризация?**
   - Speaker Diarization - это задача определения **«кто и когда говорил»**
   - Модель не распознает имена (не знает, что «SPEAKER_00» - это Иван)
   - Она только группирует голоса: «вот эти 5 фрагментов - один человек, вот эти 3 - другой»

2. **Как работает pyannote.audio?**
   - **Segmentation model** - определяет, где в аудио есть речь
   - **Embedding model** - извлекает векторное представление голоса (похоже на отпечаток пальца, но для голоса)
   - **Clustering** - группирует эмбеддинги по схожести (алгоритм кластеризации)
   - Результат: временные интервалы с метками спикеров

3. **Почему `itertracks(yield_label=True)`?**
   - Пайплайн pyannote возвращает специальный объект `Annotation`
   - `itertracks()` итерирует по временным сегментам
   - `yield_label=True` добавляет метку спикера в каждый трек
   - Формат: `(turn, track, speaker)` - где `track` нам не нужен (это внутренний ID)

4. **Почему токен нужен?**
   - Модели pyannote, особенно, `pyannote/speaker-diarization-community-1` - **gated модель**
   - Нужно зарегистрироваться на HuggingFace, принять лицензионное соглашение
   - Токен доказывает, что вы приняли условия и можете скачать веса модели
   - Без токена `Pipeline.from_pretrained()` вернет ошибку лицензионного соглашения

5. **Как это связано с другими модулями?**
   - `Diarizer` дает: `[{start, end, speaker}]`
   - `Transcriber` дает: `[{start, end, text}]`
   - `Summarizer._align_segments()` объединяет их по пересечению временных интервалов
   - Результат: `[{start, end, text, speaker}]`
