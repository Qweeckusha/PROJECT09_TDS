# Transcriber module
> `docs-ver: 1.0` | `Lib: faster-whisper` 

## Введение
Модуль отвечает за **транскрибацию аудио** - преобразование речи в текст с временными метками. Используется оптимизированная версия Whisper от OpenAI — **faster-whisper**, которая работает в 4 раза быстрее оригинала при сохранении качества.

### Ключевые особенности:
- **Асинхронная загрузка модели** - модель загружается один раз при создании экземпляра класса
- **VAD-фильтр** - автоматическое определение голосовых активностей (Voice Activity Detection)
- **Поддержка GPU/CPU** - автоматический выбор устройства вычислений
- **Квантизация** - оптимизация памяти через int8/float16


## Архитектура класса

### Инициализация модели
```python
class Transcriber:
    def __init__(self,
                 model_size: str = WHISPER_MODEL,
                 device: str = DEVICE,
                 compute_type: str = COMPUTE_TYPE):
        self.model = WhisperModel(
            model_size,
            device=device,
            compute_type=compute_type
        )
```

**Параметры инициализации:**

| Параметр | Тип | Описание                                         |
|----------|-----|--------------------------------------------------|
| `model_size` | `str` | Размер модели (больше = точнее, но медленнее)    |
| `device` | `str` | Устройство вычислений `"cuda"` (GPU) или `"cpu"` | 
| `compute_type` | `str` | Тип квантизации (оптимизация памяти) `"float16"`, `"int8"`, `"float32"`|

> [!NOTE]
> Модель загружается в память **один раз** при создании экземпляра. Повторные вызовы метода `transcribe_audio()` не перезагружают модель — это критично для производительности.


## Основной метод

### `transcribe_audio()`
Транскрибирует аудиофайл и возвращает список сегментов с текстом и временными метками.

```python
def transcribe_audio(self,
                     audio_path: str | Path,
                     language: str = "ru"
                     ) -> list[dict]:
```

**Параметры:**
- `audio_path` - путь к аудиофайлу (любой формат: `.mp3`, `.wav`, `.ogg`, `.m4a`)
- `language` - язык аудио (по умолчанию `"ru"` - русский)

**Возвращает:**
```python
[
    {"start": 0.0, "end": 5.42, "text": "Текст первого сегмента"},
    {"start": 5.42, "end": 8.15, "text": "Текст второго сегмента"},
    ...
]
```

## Принцип работы

### 1. Загрузка модели
При создании экземпляра `Transcriber()` модель загружается в память:
```python
transcriber = Transcriber()  # Модель загружена
```

### 2. Транскрибация
Метод `transcribe_audio()` выполняет:
- **VAD-фильтрацию** - отсекает тишину и неречевые звуки
- **Beam Search** - алгоритм поиска наиболее вероятной последовательности слов (`beam_size=5`)
- **Временное выравнивание** - определяет точные границы каждого сегмента

```python
segments, _ = self.model.transcribe(
    str(audio_path),
    language=language,
    beam_size=5,        # Ширина поиска (больше = точнее, но медленнее)
    vad_filter=True     # Фильтр голосовых активностей
)
```

### 3. Форматирование результата
Сырые данные от модели преобразуются в удобный формат:
```python
return [
    {
        "start": segment.start,
        "end": segment.end,
        "text": segment.text.strip()
    } for segment in segments
]
```


## Использование

### Пример вызова:
```python
from src.transcriber import Transcriber

transcriber = Transcriber()
segments = transcriber.transcribe_audio("audio.mp3", language="ru")

for seg in segments:
    print(f"[{seg['start']:.2f} - {seg['end']:.2f}] {seg['text']}")
```

### Интеграция с другими модулями:
- **`Summarizer`** вызывает `transcriber.transcribe_audio()` для получения текста
- **`CustomDiarizer`** работает параллельно для определения спикеров
- Результаты объединяются в `Summarizer._align_segments()`

## Производительность и оптимизация

### Выбор размера модели:

| Модель | VRAM | Скорость | Точность |
|--------|------|----------|--------|
| `tiny` | ~1 GB | 5        | 1      |
| `base` | ~1 GB | 4        | 2      |
| `small` | ~2 GB | 3        | 3      |
| `medium` | ~5 GB | 2        | 4      |
| `large-v3` | ~10 GB | 1        | 5      |

### Оптимизация через квантизацию:
- **`float16`** - стандарт для GPU (быстро, точно)
- **`int8`** - экономия памяти в 2 раза (для слабых GPU)
- **`float32`** - только для CPU (медленно, но стабильно)


## Ограничения и известные проблемы

1. **Длинные файлы** - файлы >1 часа могут вызвать OOM (Out of Memory). Рекомендуется разбивать на части.
2. **Фоновый шум** - VAD-фильтр помогает, но не идеален. Для зашумленных аудио рекомендуется предварительная обработка.
3. **Музыка** - модель может транскрибировать вокал в музыке как речь. Требуется дополнительная фильтрация.
